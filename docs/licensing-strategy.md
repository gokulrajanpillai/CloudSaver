# Licensing Strategy

CloudSaver currently uses the MIT License.

## Current Position

Keep the core MIT for now. The project is still earning trust, adoption, contributors, and
sponsorship. A permissive license reduces friction for users, package maintainers, and
contributors.

## Core Commitment

The following should remain open source:

- Local scan engine
- Storage dashboard
- Duplicate review
- Quarantine and restore primitives
- Local exports
- CLI and local web app

## Monetization Without Closing Core

CloudSaver can monetize through:

- Signed convenience builds
- Store distribution
- GitHub Sponsors
- Commercial support
- Business report templates
- Professional implementation help

## When To Reconsider

Consider a stronger reciprocal license, such as AGPLv3, only if:

- Hosted or embedded commercial usage creates unfair extraction.
- The project has meaningful community adoption.
- Contributors have agreed to a clear contribution policy.
- The change is announced with a migration window.

## Avoid For Now

Avoid BSL, Fair Source, SSPL, or custom source-available licenses at this stage. Those can
protect commercial interests, but they also reduce open-source credibility and may discourage
contributors before the project has momentum.

## Contributor Policy

CloudSaver uses a Developer Certificate of Origin. See [DCO.md](../DCO.md).

Before any future license change, document how contributor rights are handled and announce
the proposed change before it happens.
