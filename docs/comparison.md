# Comparison

CloudSaver is closest to visual disk analyzers and duplicate review tools, but its product
focus is narrower: local-first storage insight for local drives, synced cloud folders,
external drives, and NAS mounts.

## Positioning

| Product | Model | Strength | CloudSaver Takeaway |
| --- | --- | --- | --- |
| DaisyDisk | Paid proprietary app | Highly polished visual storage map and cleanup workflow | CloudSaver needs a more polished storage map and professional desktop packaging |
| GrandPerspective | Open source plus paid Mac App Store convenience | Lightweight treemap, saved scans, exports, Finder integration | CloudSaver can keep source open and monetize convenience builds |
| WinDirStat | Open-source/free disk analyzer | Dense treemap plus directory and extension views | CloudSaver should add folder drill-down and a category legend |
| dupeGuru | Open-source duplicate finder | Fuzzy matching, reference folders, safe deletion options | CloudSaver should improve duplicate review and protected-folder workflows |
| Czkawka | Open-source cleanup utility | Broad scan modes and cross-platform releases | CloudSaver should improve release quality before widening scope |
| BleachBit | Open-source cleaner | Preview-first cleanup and trust | CloudSaver should stay explicit about preview, quarantine, and restore |

## Differentiation

CloudSaver should not try to be every cleaner at once. The differentiator is:

- Local-first by default.
- Open-source core.
- Visual storage insight.
- Safe review workflows.
- Useful reports for individuals and small teams.
- Sponsorship and paid convenience builds instead of ads, telemetry, or data sales.

## Monetization Lessons

- Paid app-store builds can fund maintenance without closing the core.
- Commercial support can be sold around packaging, deployment, and reports.
- Restrictive relicensing should be avoided unless there is real commercial cloning risk.
- Sponsorship works best when users can see active maintenance, roadmap progress, and
  high-quality releases.
